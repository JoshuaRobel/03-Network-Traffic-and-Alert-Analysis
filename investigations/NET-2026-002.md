## Initial Triage (5 minutes)

### Alert Context
Snort flagged high-entropy DNS queries to `.xyz` domain — common indicator of DNS tunneling (encoding data in DNS queries for C2).

### Host Identification
- **Asset:** DEV-SQL-03 (development database server)
- **Owner:** Database team — contact: devops@company.com
- **Baseline:** This server has no history of DNS anomalies

---

## Investigation (20 minutes)

### DNS Query Analysis — Wireshark Filter
```
dns.qry.name contains "data-encoder" && ip.src == 192.168.67.89
```

**Sample Queries:**
```
U2FsdGVkX1.data-encoder.xyz
VGVzdGluZzE.data-encoder.xyz  
Q29uZmlndXJl.data-encoder.xyz
```

### Pattern Recognition
Queries follow pattern: `[base64-like-string].data-encoder.xyz`

**Hypothesis:** DNS tunneling for data exfiltration or C2

### Cross-Reference with Change Management
Checked CMDB and change tickets:
- **Found:** CR-2026-0842 — "Implement DNS-based health checks for DEV-SQL-03 monitoring"
- **Opened:** 2026-02-11 (yesterday)
- **Assigned:** DevOps team

### Contact with DevOps
Reached out to devops@company.com via Slack:

> **Josh:** Hey, seeing weird DNS queries from DEV-SQL-03 to `*.data-encoder.xyz`. Did you deploy something yesterday?
> 
> **DevOps:** Yes — new health check system. It encodes status data in DNS queries to bypass firewall restrictions for monitoring. Not actual tunneling, just health checks.

### Verification Requested
Asked DevOps for:
1. Architecture doc for health check system
2. List of expected DNS patterns
3. Confirmation this is not actual data exfiltration

**Received documentation** confirming legitimate monitoring tool using DNS for status reporting.

## MITRE ATT&CK Analysis

| Technique | ID | Status |
|-----------|-----|--------|
| DNS Tunneling | T1071.004 | **Not applicable** — benign tool using similar technique |

**Key Distinction:**
- **Malicious DNS tunneling:** Encodes C2 commands, exfiltrates data
- **This case:** Encodes health status only, no sensitive data, no C2 responses

---

## Decision: Benign / False Positive

**Rationale:**
1. Change ticket matches timeline (deployed day before alert)
2. Owner confirmed legitimate tool
3. No actual data exfiltration detected
4. Queries contain only status codes, not sensitive data
5. Domain owned by company infrastructure team

**Action:** Closed as false positive with documentation

## Documentation for Future Analysts

**Note to SOC:** DEV-SQL-03 uses DNS-based health monitoring via `data-encoder.xyz`. High-entropy queries are **expected**. Do not escalate unless:
- Queries from different servers
- Queries to different domains
